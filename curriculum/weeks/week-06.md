# Week 6: Automation & Workflows

## Theme
Set It and Forget It

## Tagline
*"Set It and Forget It: Building AI-Powered Automation"*

## Strategic Takeaway
> *"The best time you spend is the time you never have to spend again—automate what you can."*

## Learning Objectives
By the end of Week 6, learners will be able to:
1. Identify repetitive tasks suitable for automation
2. Design a simple automation workflow
3. Use OpenCode to generate automation scripts
4. Test and troubleshoot automated workflows

---

## In-Class Activities

### Activity 1: Automation Opportunity Audit (15 min)
- List 10 tasks you do repeatedly in your business/work
- Categorize each task:
  - Daily, weekly, or monthly
  - High, medium, or low complexity
  - High, medium, or low value
- Identify top 3 automation candidates

### Activity 2: Workflow Mapping Exercise (20 min)
- For your top automation candidate, map the workflow:
  - **Trigger**: What starts the process?
  - **Steps**: What actions happen in sequence?
  - **Output**: What is the end result?
- Draw a simple flowchart (paper or digital)
- Identify decision points and exceptions

### Activity 3: Build Your First Automation with OpenCode (35 min)
- OpenCode demo: Generating an automation script
- Prompt structure for automation:
  - **Role**: "You are an automation engineer..."
  - **Context**: "I need to automate [task] that happens when [trigger]..."
  - **Task**: "Write a script/approach that..."
  - **Format**: "Provide step-by-step instructions with code if applicable..."
  - **Constraints**: "Must work with [specific tools/platforms]..."
- Test the generated solution
- Iterate and refine

### Activity 4: Test & Troubleshoot (20 min)
- Run your automation
- Identify what works and what doesn't
- Debug common issues:
  - Trigger not firing
  - Data not passing correctly
  - Output format issues
- Document your solution

## Homework Assignment

### Task: Automation Implementation (100 points)
1. **Identify**: One repetitive task to automate
2. **Design**: Map the workflow (trigger → steps → output)
3. **Build**: Use OpenCode to create the automation
4. **Test**: Run the automation and document results
5. **Reflect**: What worked? What would you improve?

**Submission Requirements**: Word doc, Google Doc, or PDF with workflow diagram and screenshots (1-2 pages max)
