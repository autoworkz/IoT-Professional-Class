# Week 7: Final Project Development

## Theme
Bring It All Together

## Tagline
*"Bring It All Together: Your AI-Powered Solution"*

## Strategic Takeaway
> *"The true test of learning is creation—build something that solves a real problem."*

---

## Learning Objectives
By the end of Week 7, learners will be able to:
1. Define a capstone project scope
2. Apply all learned frameworks (RCTFC, forecasting, automation)
3. Build a complete AI-powered solution
4. Prepare for final presentation

---

## In-Class Activities

### Activity 1: Project Scope Definition (20 min)
- Define your capstone project:
  - **Problem**: What are you solving?
  - **Approach**: Which AI tools/frameworks will you use?
  - **Outcome**: What does success look like?
- Write a one-paragraph project summary
- Get instructor approval

### Activity 2: Design Sprint – Plan Your Solution (25 min)
- Create a project plan with:
  - Problem statement
  - RCTFC prompts needed
  - Data requirements (if any)
  - Automation opportunities (if any)
  - Timeline and milestones
- Sketch the solution architecture

### Activity 3: Build Session – Hands-On Development (35 min)
- Start building your solution:
  - Test prompts in ChatGPT/Claude/Perplexity
  - Use OpenCode for any coding needs
  - Document your process
- Instructor available for guidance

### Activity 4: Peer Feedback & Iteration (10 min)
- Share your project plan with a partner
- Receive feedback on clarity and feasibility
- Adjust scope if needed

---

## Homework Assignment

### Task: Final Project Completion (100 points)
1. **Complete** your AI-powered solution
2. **Document** your process:
   - Problem definition
   - Tools and frameworks used
   - Prompts crafted
   - Results achieved
3. **Prepare** a 5-minute presentation:
   - The problem you solved
   - Your approach and tools
   - The solution and results
   - What you learned

**Submission Requirements**: Word doc, Google Doc, or PDF (2-3 pages) + presentation slides

## Quiz (100 Points)

| Q# | Topic | Correct Answer |
|----|-------|----------------|
| 1 | First step in capstone project | Define a clear, specific problem to solve |
| 2 | Project scope best practice | Start small and achievable within the timeframe |
| 3 | Which frameworks to apply | RCTFC for prompts, forecasting for data, automation for workflows |
| 4 | Why document your process | To demonstrate learning and allow replication |
| 5 | Good presentation structure | Problem → Approach → Solution → Results → Lessons |
| 6 | How to measure project success | Does it solve the problem? Is it usable? Can it be maintained? |
| 7 | Common project pitfall | Scope creep—trying to do too much |
| 8 | Role of peer feedback | Identify blind spots and improve clarity |
| 9 | When to ask for help | When stuck for more than 15-20 minutes |
| 10 | Final project purpose | Demonstrate mastery of course concepts in a real application |
